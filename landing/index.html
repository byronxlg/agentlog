agentlog

Decision log daemon for agentic workflows

Capture agent decisions and reasoning - the missing layer between git history and LLM traces. Local-first, framework-agnostic, invisible when working. We used it to build itself.

Get Started View on GitHub

Why agent decisions get lost

When AI agents work on your codebase, critical context disappears. Git tells you what changed. LLM traces tell you what was said. Neither tells you why a decision was made, what alternatives were considered, or what failed before the final approach.

git log

Records what changed, but not why this approach was chosen over alternatives.

LLM traces

Capture the conversation, but important decisions are buried in thousands of tokens.

agentlog

Structured decision records with context, reasoning, and file associations. Auto-captured, queryable, and exportable.

How it works

agentlog runs as a lightweight daemon alongside your development tools. Agents (or you) write structured decision entries. Everything is stored locally in append-only JSONL files with a SQLite index for fast queries. With Claude Code, decisions are captured automatically - no manual logging needed. 

# Start the daemon
agentlog start

# Log a decision
agentlog write --type decision --title "Use SQLite for the index" \
  --body "JSONL for durability, SQLite for queries. Index is rebuildable." \
  --tags "architecture,storage"

# Search your decisions
agentlog query "SQLite"

# Export for a PR summary or retrospective
agentlog export --template pr --since 1d

Features

Local-first

Everything stays on your machine. No cloud services, no accounts, no network dependency. Your decisions live alongside your code.

JSONL + SQLite

Append-only JSONL logs are human-readable, git-committable, and greppable. SQLite index enables fast queries and full-text search.

Framework-agnostic

Works with any AI agent, any language, any workflow. Not tied to a specific LLM provider or development tool.

Export and templates

Export decisions as markdown, JSON, or text. Built-in templates for PR summaries, retrospectives, and handoff documents.

Python and TypeScript SDKs

Programmatic access from both languages. Write, query, export, and get context for prompts.

Claude Code integration

Automatic decision capture and context injection. Hooks run silently - invisible when working, safe to deploy anywhere.

Quickstart

1. Install

brew install byronxlg/agentlog/agentlog

2. Start the daemon

agentlog start

3. Log your first decision

agentlog write --type decision --title "Chose Postgres over MySQL" \
  --body "Better JSON support, needed for our schema-flexible data model."

See the Getting Started guide for the full walkthrough, or browse the CLI Reference for all available commands.
