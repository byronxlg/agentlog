agentlog

The missing layer between git history and LLM traces

A local-first decision log daemon for agentic workflows. Captures why decisions were made, what alternatives were considered, and what failed before the final approach.

The problem

Git tells you what changed. LLM traces tell you what was said. Neither tells you why a decision was made.

When an agent chooses approach A over approach B, that reasoning disappears. When something fails and the agent pivots, the failed attempt is lost. When assumptions are made, they go unrecorded.

agentlog captures these decisions as structured, queryable entries - so future sessions have the context they need.

How it works

# Start the daemon
$ agentlog start
agentlogd started (PID 12345)

# Log a decision
$ agentlog write --type decision \
    --title "Use SQLite for the index" \
    --body "SQLite keeps us single-binary and local-first" \
    --tags "architecture,database" \
    --files "internal/index/index.go"

# Query decisions
$ agentlog log --since 1h
2026-03-15 10:30:00 [decision] (a1b2c3d4) Use SQLite for the index

# Search by keyword
$ agentlog query "SQLite"

# See what decisions affected a file
$ agentlog blame internal/index/index.go

Features

Local-first

Everything stays on your machine. JSONL files you can read with cat, grep, and jq. SQLite index for fast queries.

Framework-agnostic

Works with any LLM framework, any agent architecture. CLI, Python SDK, and TypeScript SDK.

Five entry types

decision, attempt_failed, deferred, assumption, question - structured vocabulary for agent reasoning.

File blame

See which decisions affected a specific file. Like git blame, but for the reasoning behind changes.

Session tracking

Group related decisions within a coding session. Review an entire session's reasoning at once.

Single binary

Pure Go, no CGO, no external dependencies. One binary for the daemon, one for the CLI.

Get started

git clone https://github.com/byronxlg/agentlog.git
cd agentlog
make build
export PATH="$PWD/bin:$PATH"
agentlog start
